Раскладка init (2026-06-18): cm(.exe)+config.json в КОРНЕ проекта, данные (store.db/notes/imports/models) в подпапке memory/. Связь — Config.data_dir (путь к данным относительно config.json, дефолт "." = legacy single-folder; init пишет data_dir=<name>). Ctx::new(dir): config=dir/config.json, data под resolve_data_dir(dir, cfg.data_dir); нет config → fallback dir (legacy). resolve_data_dir (commands, pub(crate)): ""/"." → config_dir, абсолютный → как есть, относительный → join. init: data_folder=target/<name>, config в target/config.json, .gitignore в data_folder, бинарь копируется в КОРЕНЬ (skip если уже там), already_exists по data_folder. deinit: config.json из КОРНЯ, data_folder из root config.data_dir или --name. config-команда проверяет config.json до require_store. Обратная совместимость: старый config без data_dir → single-folder как раньше. Связано: [[init-no-arg-defaults]], [[init-step-output]], [[init-creates-agents-md]].
