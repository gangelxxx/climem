Decision (2026-06-19): init now ships agent onboarding in two layers so any coding agent (Claude Code, Codex, Cursor, Copilot, Kimi) understands cm.exe without a `cm help` round-trip. (1) A standalone CM_GUIDE.md at the project root — a full, self-contained, agent-agnostic manual with worked `command -> JSON output` examples; written by init::cm_guide, dropped by init::write_cm_guide, never clobbers an existing file. (2) The wired pointer blocks (init::entry_point_block) are now self-sufficient: they show the JSON each call returns (recall -> {id,kind,body}; map --query -> {name,kind,path,line,signature}; --uses -> {...,def_line}; --calls -> {calls,line,resolved}) and link to CM_GUIDE.md instead of forcing cm help. Rollback: InitManifest gained created_cm_guide (serde default=false for old manifests); deinit removes the guide iff init created it. The binary is rendered relative-and-runnable (.\cm.exe) via init::exe_command_display instead of a long absolute path. Why: models were spending an extra turn calling `cm help` to learn output shapes; embedding the shapes + a readable file removes that.
