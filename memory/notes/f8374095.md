Code review of src/ (2026-06-19) via the installed code-review-skill: clippy clean, zero unsafe. Two HIGH bugs found and FIXED in this session:

1. strip_block (src/import.rs) searched offsets in a to_lowercase() copy but sliced the original s — Unicode case folding can change byte length (e.g. U+0130 "İ" -> 3 bytes), shifting every later offset and panicking on a non-char-boundary when importing HTML with <script>/<style>. Fix: use to_ascii_lowercase() (byte-for-byte; tags are ASCII). Regression test: strip_block_preserves_offsets_with_multibyte_chars.

2. ApiEmbedder::request (src/embed/api.rs) used ureq::post with no timeout; a connectable-but-unresponsive endpoint hung the whole import/reindex loop. Fix: build a ureq::Agent with timeout_connect(10s) + timeout_read(60s).

Remaining issues (9 low) are written up under docs/issues/ (001..011, README index): config load misreports IO errors as not-found, print_line panics on broken pipe, NaN cosine breaks total sort order, redundant recall hot-path clones, export --query diverges from recall, API key not masked in endpoint, no embedding-dimension cap, recall --explain ignored with --fields.
