### Известные упрощения (by design, не баги — НЕ доделывались сознательно)

- **MCP — сознательно НЕ делаем (вырезан 2026-06-18).** Раньше `cm mcp` был заглушкой
  (`exit(2)`); ветку удалили из `main::run`. climem интегрируется как CLI (stdout JSONL /
  exit-code / `help`-контракт), а не как MCP-сервер: MCP нестабилен между клиентами и тянет
  долгоживущий сервер, что противоречит stateless single-shot архитектуре. Это **решение, а
  не недоделка** — не «восстанавливать» ветку как технический долг.
- **`[[ссылки]]` внутри код-блоков** считаются за рёбра — сканер литеральный (`graph::scan_wikilinks`, без markdown-парсинга). Рёбра дёшевы и dangling-терпимы.
- **Порядок `list`/`all` после `reindex`** = порядок чтения каталога (rowid вставки), не исходный `created`. `created_iso` хранится, но для сортировки не используется.
- **Свежесть графа:** `remember` строит рёбра только своей заметки; висячее ребро на новую цель оживает на следующем `reindex` (не мгновенно).
- **`export md`** оставлен в `<sub>`-формате (рендер из стора, без потерь); round-trip `export md → import` во frontmatter-форму не делался (экспорт — для бэкапа/ревью).

## Возможные улучшения (идеи, не обязательства)

- **Токен-эффективность.** План — [docs/token-efficiency-plan.md](../docs/token-efficiency-plan.md).
  **Фазы 0–3 реализованы (2026-06-18):** бенч-страж ([scripts/bench.py](../scripts/bench.py),
  токены + recall@k), lean-вывод `recall` (E1), мельче дефолт чанка 200/32 (C1), RRF вместо
  min-max + глубокий пул + breadcrumb + pre-filter (R1–R4), adaptive-k/`--min-score` + дефолт
  `limit 5` (K1). Решения — [notes/decisions.md](notes/decisions.md).
  **Фаза 4 (на будущее, opt-in/по росту KB):**
  - **X1 preview-then-get / `--expand`** — превью на первом проходе, тело лениво.
  - **X2 extractive-эксцерпт** (`--excerpt`) — топ-N предложений чанка дословно (офлайн).
  - **X3 дедуп при импорте** + сшивка overlap при чтении (трогать `notes` **и** `notes_fts`).
    _Частично снято миграцией:_ импорт теперь **идемпотентен per-original** — повторный
    `import` того же файла заменяет его чанки (контент-адресные id, `import::index_import`
    делает `delete_chunks_for_source` + переиндекс). Остаётся кросс-документный дедуп.
  - **X4 cross-encoder reranker** — `api`/за фичей, не дефолт.
  - **X5 RAPTOR-сводки** (`kind="summary"`, additive) — `api`-only.
- **ANN-индекс для вектора.** Сейчас `search::recall` считает косинус brute-force по всем
  `all_embeddings()` — линейно от числа заметок. При больших хранилищах — приближённый поиск.
- **Переэмбеддинг хранилища** — **DONE (2026-06-17):** `cm reindex --all` стирает
  производное и переэмбеддит всё (миграция векторов при смене провайдера). `warn_on_drift`
  теперь подсказывает именно `reindex --all`.
