# Память проекта climem

Это **единый, версионируемый источник правды** о проекте для будущих сессий модели и
людей. Здесь живёт всё, что нужно, чтобы быстро понять проект и **быстро починить баг или
внедрить фичу**, не переоткрывая уже известное.

> Папка предназначена для коммита в репозиторий (в отличие от `CLAUDE.md`, который держат
> в `.gitignore`). Сюда переезжает durable-знание: структура, сборка, решения, баги.
> На момент создания папки проект ещё не под git — память заведена заранее, чтобы расти
> вместе с кодом.

📏 **Правила ведения памяти — в [RULES.md](RULES.md).** Обязательный регламент (что куда
писать, когда обновлять, как писать, чего не делать). Прочти перед правками.

🛠 **Правила работы над кодом climem — в [rules/](rules/)** (написание по запросу,
оформление, ревью). Не путать с RULES.md.

🧭 **Как устроена эта папка и как воссоздать такую же — в
[HOWTO-BUILD-MEMORY.md](HOWTO-BUILD-MEMORY.md)** (мета-документ для другой модели).

## Что где лежит

| Файл | О чём |
|------|-------|
| [RULES.md](RULES.md) | **Свод правил** ведения памяти |
| [HOWTO-BUILD-MEMORY.md](HOWTO-BUILD-MEMORY.md) | **Мета**: как устроена память и как воссоздать с нуля |
| [rules/](rules/) | **Правила работы над кодом**: написание, оформление, ревью |
| [overview.md](overview.md) | Что за проект, стек, версии, фичи сборки, активная задача |
| [structure.md](structure.md) | Карта модулей `src/`, модель данных (SQLite), потоки recall/import |
| [build.md](build.md) | Сборка (cargo, фичи), профиль, тулчейн, дистрибуция через `init` |
| [conventions.md](conventions.md) | Грабли: stdin-тело, JSONL, `/флаг`, дрейф эмбеддера, FTS без стемминга, секреты |
| [known-issues.md](known-issues.md) | Баги: открытые (что чинить) + закрытые (как починили) |
| [roadmap.md](roadmap.md) | Точки расширения и планы (MCP-режим, локальная нейромодель, PDF) |
| [journal.md](journal.md) | Журнал сессий: что сделано и **почему** |
| [howto/](howto/) | Рецепты сквозных изменений: новая команда, провайдер эмбеддингов, формат импорта |
| [notes/](notes/) | Заметки по фичам + [архитектурные решения](notes/decisions.md) |

> `analysis/` (глубокие разборы/планы) и отдельный `deploy.md` не заведены — пока нет
> содержимого (у climem нет серверного деплоя; дистрибуция описана в [build.md](build.md)).
> Заводи их **по событию**, когда появится что положить (см. [RULES.md](RULES.md) §5).

## Связь с другими хранилищами знаний

- **Design-спека — [`../desc.md`](../desc.md)** (корень репо): концепция «память как
  исполняемый инструмент». Код ссылается на её разделы (`desc.md §N`). Это первоисточник
  «почему так»; [notes/decisions.md](notes/decisions.md) — выжимка решений со ссылками сюда.
- **`../README.md`** (корень) — пользовательская документация инструмента (контракт команд,
  быстрый старт). Не дублируй её — ссылайся.
- **`CLAUDE.md`** (если появится; gitignored, локальный) — тонкий: язык + инструкции агенту
  + ссылка сюда. Durable-инфа живёт в `memory/`.
- **Авто-память** `~/.claude/projects/.../memory/` (per-user, вне репо) — рабочий черновик
  модели; выжимка дистиллируется в [known-issues.md](known-issues.md) / [notes/](notes/).

Полный регламент — [RULES.md](RULES.md).
