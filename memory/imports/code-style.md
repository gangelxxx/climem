# Оформление кода climem

Пиши так, чтобы новый код читался как уже существующий `src/`.

## Общее

- **Rust 2021**, `cargo fmt` по умолчанию (4 пробела). Перед сдачей — `cargo clippy`, без
  новых ворнингов. Не глуши clippy без причины; осознанные исключения — точечным
  `#[allow(...)]` с коротким комментарием (как `dead_code` на методах интерфейса).
- **Модуль = одна ответственность** (см. таблицу в [../structure.md](../structure.md)).
  Новую крупную область — в свой модуль, а не в раздувание `commands.rs`.

## Комментарии и документация

- **Только по-английски** (как весь `src/`). Контент памяти — по-русски, код — по-английски.
- Каждый модуль открывается `//!`-док-комментом: что делает + ссылка на раздел `desc.md §N`,
  если решение оттуда. Держи эту привычку — она связывает код с обоснованием.
- Комментируй **почему**, а не **что**; короткими фразами по месту нетривиальной логики
  (примеры — `search.rs`, `embed/hashing.rs`).

## Ошибки

- Единый тип — `util::AppError`, алиас `util::Result<T>`. Используй `?` свободно (есть
  `From` для `rusqlite`/`io`/`serde_json`).
- Ошибки **ввода/контракта** давай через `AppError::with_hint(msg, "верный пример")` —
  пользователь/модель чинится за шаг. Внутренние — `AppError::new`.
- Не `panic!`/`unwrap` на путях с пользовательским вводом или I/O; `unwrap` допустим лишь
  там, где инвариант гарантирован локально.

## Вывод

- Данные — **JSONL** через `output::print_line` / `serde_json::json!`. Превью тел — через
  `util::preview`. Никакого человекочитаемого текста в stdout у команд данных
  (см. [../conventions.md](../conventions.md)).
- Нарратив, предупреждения, подсказки — в **stderr** (`eprintln!`).

## Зависимости

- Минимизируй. Прежде чем тащить crate — посмотри, нет ли уже своего решения (парсер
  аргументов `cli.rs`, UTC-время `util::iso_utc`). Тяжёлое — **за опциональной фичей**
  (`api`, `pdf`), с понятной ошибкой в сборке без неё.

## SQLite

- Параметры — только через `params![...]` (никакой конкатенации SQL).
- Трогаешь запись/удаление заметок — синхронизируй `notes` и `notes_fts` вручную
  (это не `content=`-таблица; см. [../conventions.md](../conventions.md)).
